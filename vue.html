Complete Vue.js Guide

What is Vue.js?

Vue.js is a progressive JavaScript framework for building user interfaces and single-page applications (SPAs).

Why Use Vue.js?

  • Simple and Flexible
  • Component-Based Architecture
  • Two-Way Data Binding
  • Reactive Data Handling
  • Ease of Integration

Reactivity System

Vue.js uses a reactivity system to track dependencies and automatically update the DOM when data changes.

Components

Components are reusable, independent blocks of code that allow you to build complex UIs.


Vue.component('hello-world', {
    template: '
Hello, World!
'
});

new Vue({
    el: '#app'
});

Directives

Directives are special tokens in the DOM that tell the library to do something to a DOM element. Examples: v-if, v-for, v-bind.

Computed Properties

Used for complex logic derived from existing data. Cached until dependencies change.


computed: {
    fullName() {
        return this.firstName + ' ' + this.lastName;
    }
}

Watchers

Used to watch and react to data changes.


watch: {
    name(newVal, oldVal) {
        console.log('Name changed from', oldVal, 'to', newVal);
    }
}

Vue Router

Vue Router is the official routing library for Vue.js, used to create and manage navigation between different views or pages in a Single Page Application (SPA). It allows developers to define different URL paths in their applications and map them to components, making it easy to build dynamic, client-side routing.

Key Features of Vue Router

  • Nested Routes: Supports nested routing, allowing complex UI structures with parent-child relationships.
  • Dynamic Routing: Define routes that include dynamic segments, like /user/:id, where :id is a variable.
  • Route Guards: Provides navigation guards to control access to certain routes based on authentication or other conditions.
  • History Mode: Uses the HTML5 History API to create clean URLs without hash symbols (#).
  • Lazy Loading: Supports asynchronous component loading to optimize performance by splitting code into smaller chunks.
  • Named Routes & Views: Allows assigning names to routes and rendering multiple views simultaneously.
  • Scroll Behavior Control: Handles how the scroll position is managed when navigating between routes.

    • Installation

    If you’re using Vue 3:

    
                npm install vue-router@next

    If you’re using Vue 2:

    
                    npm install vue-router

    Basic Example (Vue 3)

  • Creating a Router File (router/index.js):
    • 
                import { createRouter, createWebHistory } from 'vue-router';
                import Home from '../views/Home.vue';
                import About from '../views/About.vue';

                const routes = [
                { path: '/', name: 'Home', component: Home },
                { path: '/about', name: 'About', component: About }
                ];

                const router = createRouter({
                history: createWebHistory(),
                routes
                });

                export default router;

    Using Router in a Vue App (main.js):

    
                import { createApp } from 'vue';
                import App from './App.vue';
                import router from './router';

                const app = createApp(App);
                app.use(router);
                app.mount('#app');

    Defining Navigation Links (App.vue):

    <template>
                <div>
                  <nav>
                    <router-link to="/">Home</router-link>
                    <router-link to="/about">About</router-link>
                  </nav>
                  <router-view></router-view>
                </div>
              </template>

    How It Works

  • The router-link component is used to navigate between pages without refreshing the page.
  • The router-view component acts as a placeholder where matched components are rendered based on the current URL.

    • Why Use Vue Router?

  • Makes URL handling and navigation straightforward in Vue applications.
  • Provides robust tools for authentication, access control, and page transitions.
  • Improves performance with lazy loading and code-splitting.
