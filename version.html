Version Control - Git GitHub

Introduction to Version Control

Version control is a system that helps developers track and manage changes to code over time. Git is a popular version control tool, and GitHub is a platform for hosting and sharing Git repositories.

Why Use Version Control?

  • Collaboration: Allows multiple developers to work on a project simultaneously.
  • Backup: Provides a secure backup of the entire codebase.
  • Versioning: Tracks changes, making it easy to revert to previous versions.
  • Branching: Facilitates experimentation with new features without breaking the main codebase.

Basic Git Commands


// Initializing a Git Repository
git init

// Checking the status of your repository
git status

// Adding files to staging area
git add .

// Committing changes
git commit -m "Initial commit"

// Pushing changes to GitHub
git push -u origin main

// Cloning a repository from GitHub
git clone https://github.com/username/repository.git

// Creating a new branch
git branch feature-branch

// Switching to a branch
git checkout feature-branch

// Merging branches
git merge feature-branch

Working with GitHub

GitHub allows developers to host, share, and collaborate on Git repositories. Some basic operations include:

  • Create a Repository: Sign in to GitHub and click "New Repository" to start a new project.
  • Push Code to GitHub: After creating your local repository, use git push to upload code to GitHub.
  • Create a Pull Request: Use pull requests to suggest code changes and request reviews from collaborators.
  • Issues & Discussions: GitHub provides tools for tracking bugs and collaborating through discussions.

Example Git Workflow


// Step 1: Clone the Repository
git clone https://github.com/username/repo.git

// Step 2: Create a New Branch
git checkout -b feature-branch

// Step 3: Make Changes & Stage Files
git add .

// Step 4: Commit Changes
git commit -m "Implemented a new feature"

// Step 5: Push to GitHub
git push -u origin feature-branch

// Step 6: Create a Pull Request on GitHub

Branching and Merging

Branching allows you to create separate lines of development. Merging integrates changes from different branches.


// Create a New Branch
git branch new-feature

// Switch to the Branch
git checkout new-feature

// Merge Branches
git checkout main
git merge new-feature

Collaborating with GitHub

GitHub allows teams to work together effectively using features like pull requests, issues, and forks.

Handling Conflicts

Conflicts occur when changes are made to the same part of a file by different contributors. Resolving conflicts involves editing files and committing the resolved versions.


// View Conflicted Files
git status

// Edit Files to Resolve Conflicts
// Stage Resolved Files
git add .

// Commit Resolved Changes
git commit -m "Resolved merge conflicts"

Git Best Practices

  • Make frequent commits with descriptive messages.
  • Use branching for developing new features.
  • Regularly push changes to remote repositories.
  • Review code changes before merging.
